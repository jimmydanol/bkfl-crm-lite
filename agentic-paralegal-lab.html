What becomes possible

The power is controlled repetition.

The agent does not “answer the case.” It keeps a durable objective, uses bounded tools, reacts to events, and knows when authority belongs to a person.

01 / ObserveRead the newest case state

Consume a locked Intake Package, document bundle metadata, corrections, and timeline events without silently overwriting history.

02 / GroundCite every finding

Attach the exact fake field, bundle count, event, and configured rule so staff can verify why the agent stopped.

03 / Act narrowlyDraft and route work

Propose questions, Task payloads, and review queues while external messages and firm rulings remain human-owned.

04 / Re-loopReact to new evidence

When a correction or upload arrives, reopen only affected gates, rerun relevant checks, and preserve the event trail.

05 / StopEscalate uncertainty

Route legal significance, waivers, attorney-only warnings, and Stage 4 transition to the authorized reviewer.

Dinner walkthrough

A 90-second proof for Matt

Start with Riley to show productive looping. End with Taylor to show the more important capability: knowing when to stop.

Say and click

  1. Load Riley Chen. “This is a fake locked Intake Package, not a chatbot prompt.”
  2. Run to human checkpoint. Show the missing fact, partial bank bundle, employer-name mismatch, correction event, and counseling-policy question—with evidence.
  3. Inspect the draft. The agent prepares neutral follow-up work but cannot send it.
  4. Inject a fake answer and upload. Run the loop again and show only affected gates being re-checked.
  5. Switch role to Attorney. Resolve policy/warnings, then show “eligible under configured rules”—not an autonomous Stage 4 transition.
  6. Load Taylor Brooks. Show attorney-only transfer/property conflicts and the agent stopping instead of inventing legal consequences.

Non-negotiable guardrails

  • Synthetic data only; no debtor PII on GitHub Pages.
  • GitHub Pages demonstrates behavior, not production security.
  • No API key, model credential, or confidential prompt in browser code.
  • Uploaded text is untrusted data and cannot change agent instructions.
  • No legal advice, chapter selection, exemption determination, or filing-ready petition.
  • No autonomous send, waiver, document approval, attorney approval, or stage transition.
  • Locked Intake answers are versioned and attributed; never silently overwritten.
